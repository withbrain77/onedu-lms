# ONEDU 운영 전 1~5단계 점검표

이 문서는 정식 오픈 전 개발/운영자가 확인해야 하는 5개 영역을 정리한다.

## 1. 운영 안정화

- `python manage.py check` 통과
- `python manage.py test` 통과
- `python manage.py makemigrations --check --dry-run` 통과
- 운영에서는 `python manage.py ops_readiness` 실행
- 서버에서는 가능하면 다음 명령으로 확인

```bash
sudo -n /usr/local/sbin/onedu-ops readiness
sudo -n /usr/local/sbin/onedu-ops health
```

확인할 항목:

- 최근 24시간 메일 발송 실패
- HLS 변환 실패 또는 장시간 변환 중 작업
- 최근 24시간 접속 주의 기록
- 유료 강의 입금 확인 대기
- 재수강 승인 대기
- media/private_media/log 경로 쓰기 가능 여부

## 2. 관리자 업무 화면

수강 신청 관리에서 사용할 수 있는 빠른 작업:

- `선택한 유료 신청 입금 확인 처리`
- `선택한 신청 입금 확인 후 승인(기본 기간)`

메일 발송 로그에서 사용할 수 있는 빠른 작업:

- `선택한 실패/대기 메일 재발송`

운영 방식:

- 유료 강의는 입금 확인 후 승인한다.
- 일괄 승인 액션은 수강 시작일이 비어 있으면 오늘 날짜를 사용한다.
- 수강 종료일이 비어 있으면 강의의 기본 수강 기간을 사용한다.
- 탈퇴 요청 또는 비활성 계정은 승인하지 않고 건너뛴다.

## 3. 수강생 안내/알림

수강생 내 강의실에서 확인할 수 있는 안내:

- 신청 접수 후 입금/승인 대기 안내
- 입금 계좌 안내
- 수강 중인 강의의 마감 전 학습 안내

메일 알림:

- 유료 강의 신청 시 관리자 알림
- 승인 시 수강생 알림
- 수강 종료 7일 전 수강생 알림
- 탈퇴 요청 시 관리자 알림

메일 실패 시:

1. 관리자 `메일 발송 로그`에서 실패 원인을 확인한다.
2. SMTP 설정을 수정한다.
3. 실패 로그를 선택해 `선택한 실패/대기 메일 재발송`을 실행한다.

## 4. 부정 이용 의심 리포트

회원 관리의 `학습 리포트`에서 확인한다.

지표:

- 총 시청 시간
- 반복 시청 배수
- 접속 IP 수
- 접속 기기 수
- 최근 접속 기록
- 주의 접속 건수
- 자료 다운로드 이력

주의:

- 반복 시청 배수와 IP/기기 수는 참고 지표다.
- 바로 차단하지 말고, 수강생 문의 또는 내부 확인 후 판단한다.

## 5. 백업/복구 점검

DB 백업 생성:

```bash
sudo -n /usr/local/sbin/onedu-ops backup-db
```

백업 상태 확인:

```bash
sudo -n /usr/local/sbin/onedu-ops backup-check
```

반드시 백업해야 하는 경로:

- PostgreSQL DB dump
- `data/media`
- `data/private_media`
- `.env`

복구 리허설:

- 정식 오픈 전 최소 1회는 별도 테스트 폴더 또는 테스트 NAS에서 복구 절차를 확인한다.
- 운영 DB 볼륨에 직접 복구 명령을 실행하기 전에는 반드시 최신 백업을 별도로 보관한다.
