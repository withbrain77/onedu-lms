# 개발 단계별 체크리스트

## 1. 기본 원칙

- 코드 구현 전 설계 문서를 먼저 확정한다.
- 초기 MVP에서는 Django template와 Bootstrap 5를 사용한다.
- Django Admin을 적극 활용하되, 운영 빈도가 높은 업무는 `dashboard`로 분리한다.
- 권한 검증과 수강 기간 검증은 공통 서비스로 분리한다.
- UI는 기본 브라우저 스타일로 방치하지 않는다.
- 수강생 화면은 모바일 반응형을 필수로 검증한다.

## 2. 1단계: 프로젝트 기반 구조

목표:

- Django 프로젝트 생성
- 앱 구조 생성
- 환경별 settings 분리
- PostgreSQL 연결 준비
- Bootstrap 5 기반 base template 준비

체크리스트:

- [ ] `config` 프로젝트 생성
- [ ] `accounts`, `courses`, `lessons`, `enrollments`, `progress`, `quizzes`, `certificates`, `notices`, `dashboard`, `core` 앱 생성
- [ ] custom User 모델 적용
- [ ] settings `base`, `local`, `production` 분리
- [ ] 환경변수 로딩 구조 추가
- [ ] 기본 URL 구조 연결
- [ ] `base.html`, navbar, messages, page header partial 생성
- [ ] Bootstrap 5 적용
- [ ] 공통 CSS 파일 생성

완료 기준:

- 로컬에서 Django 서버가 실행된다.
- 기본 페이지가 Bootstrap 5 스타일로 표시된다.
- 관리자와 수강생 메뉴 분리 기준이 잡혀 있다.

체크리스트:

- [ ] 강의 상세에 차시 목록과 완료 여부 표시

완료 기준:

체크리스트:

- 수강생별 이력, 진도, 시험, 수료 상태를 한 화면에서 확인할 수 있다.

## 11. 통합 리팩터링 기준

메인 에이전트는 각 단계 후 다음을 확인한다.

- 같은 권한 로직이 중복 구현되지 않았는가
- template에서 지나치게 복잡한 조건문을 사용하지 않았는가
- dashboard와 Django Admin의 역할이 중복되지 않는가
- UI 컴포넌트 스타일이 일관적인가
- 모델과 서비스의 책임이 분리되어 있는가
- 테스트 가능한 구조인가
