# MVP 구현 계획과 추후 확장 기능

## 1. MVP 목표

MVP의 목표는 회사 내부에서 실제로 운영 가능한 승인형 온라인 강의 사이트를 빠르게 만드는 것이다.

MVP는 다음에 집중한다.

- 수강생 계정
- 강의와 차시 관리
- 수강 신청과 관리자 승인
- 수강 기간 관리
- 승인된 수강생만 영상 시청
- 영상 진도 저장
- 기본 시험
- 수료 판정
- 수료증 다운로드
- 관리자 운영 화면
- Docker Compose 배포

## 2. MVP 포함 기능

### 계정

- 회원가입
- 로그인, 로그아웃
- 내 정보 수정
- 관리자와 수강생 권한 분리
- 관리자 수강생 계정 생성
- 관리자 비밀번호 초기화

### 강의

- 강의 생성, 수정, 삭제
- 강의 설명
- 썸네일
- 공개 여부
- 강의 목록
- 강의 상세

### 차시와 영상

- 강의별 차시 생성
- 차시 순서 관리
- 영상 업로드
- 영상 공개, 비공개
- 권한 확인 후 영상 제공
- 동적 워터마크
- 모바일 영상 화면

### 수강

- 수강 신청
- 승인, 반려
- 시작일, 종료일 지정
- 기간 연장
- 승인된 강의 확인
- 내 강의실
- 기간 만료 후 재수강 신청

### 진도

- 영상별 마지막 시청 위치
- 진도율
- 시청 완료 여부
- 관리자 진도 확인

### 시험

- 객관식 문제
- 주관식 문제
- 합격 점수 설정
- 시험 응시
- 점수 확인
- 관리자 결과 확인

### 수료증

- 수료 조건 확인
- 수료증 발급
- 수료증 다운로드
- 관리자 발급 관리

### 공지

- 공지사항 관리
- 전체 또는 강의별 공지

## 3. MVP에서 단순화할 항목

초기 구현 속도를 위해 다음은 단순화한다.

- 영상 트랜스코딩 자동화는 제외
- HLS 스트리밍은 제외
- 고급 DRM은 제외
- 이메일 인증은 후순위
- SMS 알림은 제외
- 결제 기능은 제외
- SCORM/xAPI는 제외
- 복잡한 통계 대시보드는 제외
- 다국어는 제외
- 실시간 채팅 또는 Q&A는 제외

## 5. MVP 완료 기준

수강생 기준:

- 회원가입과 로그인이 가능하다.
- 공개 강의를 보고 수강 신청할 수 있다.
- 승인 전에는 영상을 볼 수 없다.
- 승인 후 수강 기간 내에만 영상을 볼 수 있다.
- 내 강의실에서 상태, 기간, 진도율을 확인할 수 있다.
- 영상을 보고 진도가 저장된다.
- 시험을 응시하고 점수를 확인할 수 있다.
- 수료 조건 충족 시 수료증을 다운로드할 수 있다.
- 기간 만료 시 재수강 신청을 할 수 있다.

관리자 기준:

- 강의와 차시를 관리할 수 있다.
- 영상을 업로드하고 공개 여부를 관리할 수 있다.
- 수강 신청을 승인 또는 반려할 수 있다.
- 수강 시작일과 종료일을 지정할 수 있다.
- 수강생별 진도, 시험 결과, 수료 여부를 확인할 수 있다.
- 재수강 신청을 승인 또는 반려할 수 있다.
- 공지사항을 관리할 수 있다.

운영 기준:

- Docker Compose로 실행 가능하다.
- PostgreSQL 데이터를 유지한다.
- static, media, private video storage가 분리되어 있다.
- 운영 환경변수를 사용한다.
- DB와 영상 파일 백업 절차가 있다.

## 6. 주요 리스크와 대응

| 리스크 | 대응 |
| --- | --- |
| 영상 직접 URL 노출 | private media와 `X-Accel-Redirect` 사용 |
| 권한 로직 중복 | `core/services/access.py`로 통합 |
| 모바일 영상 화면 깨짐 | 영상 화면을 우선 반응형 QA 대상으로 지정 |
| 전체화면 워터마크 미표시 | video 단독이 아닌 wrapper fullscreen 사용 |
| Django Admin만으로 운영 불편 | 승인, 진도, 수료 업무는 dashboard로 분리 |
| NAS 경로 종속 | 환경변수로 volume 경로 분리 |
| 개인정보 노출 | 필드 최소화, 관리자 화면 접근 제한, 로그 점검 |

- 강의별 수료 통계
- 수강생별 리포트
- 감사 로그
