# UI/UX 설계안

## 1. 디자인 방향

전체 UI는 깔끔한 업무용 LMS 스타일로 설계한다. 장식보다 정보 구조와 업무 흐름을 우선한다.

기준:

- Django template 기반
- Bootstrap 5 사용
- 흰색과 밝은 회색 배경
- 파란색 계열 포인트 컬러
- 충분한 여백과 명확한 상태 표시
- 버튼, 카드, 테이블, 폼, 배지 스타일 통일
- 수강생 화면은 모바일 반응형 필수
- 관리자 화면은 데스크톱 업무 환경 우선

## 2. 공통 템플릿 구조

```text
templates/
  base.html
  partials/
    navbar.html
    messages.html
    page_header.html
    empty_state.html
    pagination.html
    confirm_modal.html
    status_badge.html
  accounts/
  courses/
  classroom/
  lessons/
  quizzes/
  certificates/
  dashboard/
```

`base.html` 기본 구성:

```text
상단 네비게이션
페이지 제목 영역
메시지 영역
본문 컨테이너
공통 모달 영역
공통 스크립트
```

## 3. 공통 UI 컴포넌트

### 상단 네비게이션

표시 항목:

- 서비스명 또는 로고
- 수강생 메뉴: 내 강의실, 강의 목록, 수료증
- 관리자 메뉴: 대시보드, 수강 신청, 강의 관리, 수강생 관리
- 로그인 사용자명
- 로그아웃 버튼

메뉴는 역할에 따라 분리한다.

### 페이지 제목

각 화면 상단에는 현재 위치와 주요 액션을 보여준다.

예시:

```text
내 강의실
수강 중인 강의와 신청 상태를 확인합니다.
[강의 목록 보기]
```

설명 문구는 짧게 유지하고, 다음 행동은 버튼으로 보여준다.

### 메시지

Bootstrap alert 사용:

```text
success
warning
danger
info
```

예시:

- 수강 신청이 접수되었습니다.
- 수강 기간이 종료되었습니다.
- 접근 권한이 없습니다.

### 빈 데이터 화면

빈 화면은 단순 공백으로 두지 않는다.

예시:

```text
수강 중인 강의가 없습니다.
[강의 목록 보기]
```

### 로딩과 처리 중 상태

폼 제출 버튼은 처리 중 상태를 표시한다.

예시:

```text
승인하기
처리 중...
```

진도 저장은 작은 텍스트 또는 상태 아이콘으로만 표시한다.

```text
최근 진도 저장됨
```

## 4. Bootstrap 5 스타일 기준

버튼:

| 목적 | 클래스 |
| --- | --- |
| 주요 행동 | `btn btn-primary` |
| 보조 행동 | `btn btn-outline-secondary` |
| 성공 행동 | `btn btn-success` |
| 위험 행동 | `btn btn-outline-danger` |
| 작은 테이블 행동 | `btn btn-sm` 조합 |

배지:

| 상태 | 클래스 |
| --- | --- |
| 승인됨 | `badge text-bg-success` |
| 신청 대기 | `badge text-bg-warning` |
| 반려 | `badge text-bg-danger` |
| 종료 | `badge text-bg-secondary` |
| 진행 중 | `badge text-bg-primary` |

진도율:

```text
progress
progress-bar
```

테이블:

```text
table table-hover align-middle
```

폼:

```text
form-label
form-control
form-select
invalid-feedback
```

카드:

- 강의 카드
- 관리자 지표 카드
- 빈 상태 카드

카드 중첩은 피한다.

## 5. 수강생 화면 UX

### 로그인 후 이동

수강생은 로그인 후 바로 `/classroom/`으로 이동한다.

### 내 강의실

구성:

```text
탭 또는 섹션
  수강 중
  승인 대기
  종료됨
```

강의 카드 표시 정보:

```text
강의명
수강 상태 배지
수강 시작일
수강 종료일
남은 기간
진도율 progress bar
이어보기 버튼
강의 보기 버튼
```

상태별 주요 버튼:

| 상태 | 주요 버튼 |
| --- | --- |
| 승인됨, 기간 내 | 이어보기 |
| 승인 대기 | 신청 상태 확인 |
| 반려 | 강의 목록 보기 |
| 종료 | 재수강 신청 |
| 수료 가능 | 수료증 다운로드 |
| 시험 필요 | 시험 응시 |

### 강의 상세

표시 정보:

- 강의명
- 수강 상태
- 수강 기간
- 전체 진도율
- 차시 목록
- 각 차시 완료 여부
- 마지막 시청 위치
- 시험 상태
- 수료증 상태

차시 목록은 완료된 항목과 미완료 항목이 시각적으로 구분되어야 한다.

### 영상 시청

데스크톱:

```text
왼쪽 넓은 영역: 영상 플레이어
오른쪽 보조 영역: 차시 목록, 진도율, 다음 강의 버튼
```

모바일:

```text
영상 플레이어
진도율과 다음 강의 버튼
차시 목록
```

영상 영역이 페이지의 중심이 되도록 한다. 보조 정보가 영상을 밀어내지 않게 한다.

## 6. 영상 워터마크 UX

표시 내용:

```text
수강생 이름
사용자 ID 일부
현재 시각 또는 세션 식별 텍스트
```

스타일:

- 반투명
- 영상 시청을 방해하지 않는 크기
- 흰색 또는 밝은 회색 텍스트에 약한 그림자
- `pointer-events: none`

동작:

- 일정 시간마다 위치 변경
- 중앙, 좌상단, 우상단, 좌하단, 우하단 등 제한된 위치 사용
- 화면 밖으로 나가지 않게 위치 제한

전체화면 기준:

- video 요소가 아니라 player wrapper를 전체화면 처리한다.
- wrapper 내부에 watermark layer를 둔다.

## 7. 관리자 화면 UX

관리자 화면은 데스크톱 업무 효율을 우선한다. 많은 정보를 한 번에 비교할 수 있게 테이블 중심으로 구성한다.

### 관리자 대시보드

상단 지표:

- 전체 수강생 수
- 승인 대기 신청 수
- 진행 중 수강 수
- 수료자 수
- 재수강 신청 수

하단 영역:

- 최근 수강 신청
- 최근 시험 응시 결과
- 수료 대기 또는 수료 가능 대상
- 진도 낮은 수강생

### 수강 신청 관리

필터:

- 검색어
- 강의
- 상태
- 신청일 범위

테이블 컬럼:

```text
신청일
수강생
강의명
상태
희망 또는 지정 시작일
지정 종료일
승인/반려 액션
```

승인 처리:

- 시작일 date input
- 종료일 date input
- 승인 버튼
- 확인 모달

반려 처리:

- 반려 사유 입력
- 확인 모달

### 수강생 상세

한 화면에서 확인할 정보:

- 기본 정보
- 마지막 접속일
- 계정 상태
- 수강 이력
- 강의별 진도율
- 영상별 시청 여부
- 시험 결과
- 수료증 발급 여부

### 강의 관리

확인 정보:

- 강의 공개 여부
- 썸네일
- 차시 수
- 영상 업로드 여부
- 영상 공개 여부
- 차시 순서

CRUD 자체는 초기 MVP에서 Django Admin을 활용할 수 있다. 커스텀 대시보드는 운영자가 자주 확인하는 요약과 처리 화면 위주로 만든다.

## 8. 폼 UX

폼 기준:

- 필수 입력값 표시
- 입력 오류를 필드 바로 아래 표시
- 서버 오류는 상단 alert와 필드 오류를 함께 사용
- 날짜 입력은 `input type="date"` 사용
- 긴 설명 입력은 textarea 사용
- 제출 후 중복 클릭 방지

## 9. 모바일 기준

필수 확인 화면:

- 로그인
- 회원가입
- 강의 목록
- 내 강의실
- 강의 상세
- 영상 시청
- 시험 응시
- 수료증 목록
- 재수강 신청

모바일 원칙:

- 테이블 대신 카드 또는 스택 레이아웃 사용
- 버튼은 터치 가능한 크기 확보
- 영상과 차시 목록은 세로 배치
- 긴 강의명은 줄바꿈 처리
- progress bar와 배지가 화면 밖으로 밀리지 않게 처리

## 10. 접근성과 사용성

기본 기준:

- 버튼 텍스트는 행동을 명확히 표현한다.
- 색상만으로 상태를 구분하지 않는다.
- 상태 배지와 텍스트를 함께 사용한다.
- 폼 label을 생략하지 않는다.
- 모달에는 명확한 취소 버튼과 확인 버튼을 둔다.
- 위험 작업은 한 번 더 확인한다.
